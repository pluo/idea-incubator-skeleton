# Decision 2: Name the Project idea-incubator-skeleton

## Decision

Name the public GitHub project `idea-incubator-skeleton`.

## Context

This project is intended to become public, so the name should be easy to
understand and easy to find on GitHub.

The project is a reusable idea incubation skeleton. It helps preserve
public-safe provenance from the first intuition through notes, AI-assisted
sessions, decisions, specs, research, and optional implementation.

The name should describe the project clearly, avoid unnecessary ambiguity, and
remain easy to search.

## Options Considered

- `idea-incubator-skeleton`: clear, searchable, and aligned with the project's
  early-stage incubation purpose.
- `idea-incubation-skeleton`: accurate, but more abstract and less object-like.
- `idea-repo-skeleton`: abandoned because "idea repo" can be confused with an
  IntelliJ IDEA repository.

## Rationale

`idea-incubator-skeleton` is concrete enough to describe the reusable structure
and broad enough to cover notes, sessions, decisions, specs, research, and
eventual code.

It keeps the useful "idea" and "skeleton" terms, but replaces the ambiguous
"repo" phrasing with "incubator", which better describes how early ideas are
captured and developed.

## Consequences

- The public repository should be named `idea-incubator-skeleton`.
- Public documentation should use `idea-incubator-skeleton` as the project name.
- The copyable template directory should be named `idea-incubator-skeleton/`.
- The current local development checkout can remain wherever it already lives.
- Future naming choices should favor clarity and searchability over novelty.
